# Aydan Automates — Brand Kit

Use this document as a reference when creating any branded deliverables (PDFs, slides, social graphics, documents, etc.) for Aydan Automates.

---

## Brand Identity

**Brand Name:** Aydan Automates
**Tagline:** AI Coach, Consultant, & Automation Specialist
**Founder:** Aydan (the AI Guy)
**Voice:** Direct, no-nonsense, technically literate but accessible. Speaks like a knowledgeable peer — not a corporate entity. Avoids jargon for jargon's sake. Confident without being salesy.

## Color Palette

### Primary Colors
| Role | Hex | Description |
|------|-----|-------------|
| Background | `#0A0A0A` | Near-black, primary background |
| Light Background | `#111111` | Slightly elevated surfaces / alternate sections |
| Card Background | `#1A1A1A` | Cards, containers, content blocks |
| Card Hover | `#222222` | Hover/active state for cards |
| Border | `#2A2A2A` | Subtle dividers and borders |

### Text Colors
| Role | Hex | Description |
|------|-----|-------------|
| Primary Text | `#F5F5F5` | Headlines, primary copy |
| Body Text | `#CCCCCC` | Paragraph body copy |
| Secondary Text | `#AAAAAA` | Subheadings, supporting copy |
| Muted Text | `#888888` | Labels, captions, metadata |
| Dim Text | `#767676` | Smallest annotations (WCAG AA compliant) |

### Summary
The palette is dark-mode-first with a near-black base and a monochromatic gray scale for text hierarchy. Think: dark terminal aesthetic, clean and minimal.

## Typography

### Font Families
| Font | Type | Usage |
|------|------|-------|
| **PT Mono** | Monospace | Headlines, section labels, buttons/CTAs, navigation, card titles, captions |
| **DM Sans** | Sans-serif | Body copy, paragraphs, general reading text |

Both are available via [Google Fonts](https://fonts.google.com/).

### Font Weights
- **PT Mono:** Regular (400) only
- **DM Sans:** Light (300), Regular (400), Medium (500), SemiBold (600), Bold (700)

### Type Scale & Styles

**Hero Headline**
- Font: PT Mono, 700 weight
- Size: Large (roughly 2.4rem–4.2rem depending on viewport)
- Line height: 1.08
- Color: `#F5F5F5`

**Section Headlines**
- Font: PT Mono, 700 weight
- Size: Medium-large (roughly 1.85rem–3rem depending on viewport)
- Line height: 1.15
- Color: `#F5F5F5`

**Section Labels** (small tag above headlines)
- Font: PT Mono
- Size: Small (0.8rem)
- Style: ALL CAPS, letter-spacing 0.15em
- Color: `#888888`
- Often displayed inside a pill/tag with a `#2A2A2A` border and semi-transparent dark background

**Body Copy**
- Font: DM Sans, 400 weight
- Size: ~1.05rem–1.15rem
- Line height: 1.7
- Color: `#CCCCCC`

**Buttons / CTAs**
- Font: PT Mono
- Size: Small (0.85rem)
- Style: ALL CAPS, letter-spacing 0.1em
- Primary state: White text on near-black, or near-black text on white
- Hover state: Outlined/transparent with white text

**Small/Meta Text**
- Font: PT Mono
- Size: 0.7rem–0.8rem
- Style: ALL CAPS, letter-spacing 0.12em–0.18em
- Color: `#888888` or `#767676`

## Visual Style

### Overall Aesthetic
- **Dark-mode-first** — all backgrounds are near-black or very dark gray
- **Minimal and clean** — generous whitespace, no clutter
- **Monospace-forward** — PT Mono gives everything a technical, builder-oriented feel
- **Subtle texture** — a faint grain/noise overlay adds warmth and depth to backgrounds
- **Sharp edges** — no border-radius on cards or buttons (rectangular, not rounded)
- **Restrained motion** — subtle fade-ups on scroll, no flashy animations

### Cards & Containers
- Background: `#1A1A1A`
- Border: 1px solid `#2A2A2A`
- Padding: Generous (32px–40px)
- No border-radius (square corners)
- On hover: background shifts to `#222222`, border lightens to `#333333`, slight upward lift

### Borders & Dividers
- Color: `#2A2A2A`
- Weight: 1px
- Used subtly to separate sections and define card edges

### Spacing Philosophy
- Sections: 100–120px vertical padding
- Content max-width: 1120px
- Comfortable margins, nothing feels cramped
- Content blocks are left-aligned (not centered text)

---

## Tone & Copy Style

- **Direct and conversational** — writes like talking to a smart friend, not a boardroom
- **Short sentences.** Punchy. Gets to the point.
- **Uses "you" and "I"** — personal, not corporate ("I help you..." not "We provide solutions for...")
- **Technically confident** — comfortable naming tools and systems without over-explaining
- **No hype or buzzwords** — avoids "revolutionary," "game-changing," "synergy"
- **Occasional dry humor** — keeps things human
- **Lowercase-leaning** in casual contexts; ALL CAPS only for labels and buttons

## Logo & Imagery

- **Logo:** Custom SVG mark used alongside the wordmark "AYDAN AUTOMATES" in DM Sans Bold, all caps, wide letter-spacing (0.2em)

- **No stock photography** — real photos only

## Do's and Don'ts

### Do
- Use the dark background palette as the default
- Keep text hierarchy clear with the monochromatic gray scale
- Use PT Mono for anything that should feel "branded" (headlines, labels, CTAs)
- Use DM Sans for anything that should feel readable (paragraphs, long-form)
- Keep layouts clean with generous whitespace
- Use sharp/square corners on containers and buttons
- Left-align text content

### Don't
- Use light/white backgrounds as the primary canvas
- Use rounded corners or pill-shaped buttons
- Use decorative or script fonts
- Center-align body text
- Use gradients, shadows, or glossy effects
- Use stock photos or generic illustrations
- Write in third person or corporate-speak
